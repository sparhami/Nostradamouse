nmouse-rule

Description

Consists of one or more <nmouse-trigger> elements that dictate when an import should be loaded

A rule should be placed in the same shadow root as any elements it will act on

Attributes

  • src The path to load
  • selector A CSS selector, used with event-type triggers
  • el A CSS selector, used with proximity-type triggers

nmouse-trigger

Description

Defines a trigger to load a rule on. There are currently 3 types of triggers:

  • Event-based: uses event delegation and a selector to determine if an event on some element should trigger the rule
  • Proximity-based: based on distance from the cursor to a given element
  • Element added: triggers when an element of a specified tag name is added to the document

Attributes

  • type One of 'added', 'proximity' or an event name (e.g. 'click')
  • tagName A tag name (e.g. x-my-component), used with type="added"
  • distance A distance, in pixels, used with type="proximity"

Example

<nmouse-rule src="components/x-foo/x-foo.html" el="#some-element"> <nmouse-trigger type="proximity" distance="100"> </nmouse-trigger> </nmouse-rule> <nmouse-rule src="components/x-bar/x-bar.html" selector=".some-class"> <nmouse-trigger type="mouseenter"></nmouse-trigger> <nmouse-trigger type="focus"></nmouse-trigger> </nmouse-rule> <nmouse-rule src="components/x-baz/x-baz.html"> <nmouse-trigger type="added" tagName="x-baz"></nmouse-trigger> </nmouse-rule>

nmouse-deps

Description

Used to load dependency information upfront. Typically, you will want to generate the dependency information and bundle it into the initial payload using something like Vulcanize.

See: NostradamouseDeps for generating the necessary dependency information

Example

<nmouse-deps> <nmouse-dep data-id="0" src="components/x-foo/x-foo.html" deps="1 2"> </nmouse-dep> <nmouse-dep data-id="1" src="components/x-bar/x-bar.html" deps="2"> </nmouse-dep> <nmouse-dep data-id="2" src="components/x-baz/x-baz.html"> </nmouse-dep> </nmouse-deps>
