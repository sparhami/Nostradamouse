On clicking (or focusing) the input box, the definition for the <x-stateautosuggest> is loaded. The <lazy-autosuggest> component that it uses makes sure to have the correct state (e.g. open, fetch suggestions based on the input value) when the component definition is loaded and the elmeent is upgraded.

By passing the <input type="text" /> component to the <x-stateautosuggest> component (which in turn passes it down to the <lazy-autosuggest>, the element can appear on the screen as soon as the page is loaded.

The suggestions (U.S. states and capitols) are contained within the <x-stateautosuggest> component, but there is a 100ms delay before showing them so that you might get a better feel for how responsive the component will be when the user firsts loads the page and starts typing into it.
